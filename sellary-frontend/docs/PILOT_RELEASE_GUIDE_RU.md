# Pilot Release Guide

## Цель

Этот документ нужен для первого пилотного запуска Sellary MVP в retail-сценарии.

## 1. Что входит в пилот

- Розничная касса
- Товары и остатки
- Поставщики
- Закупки и приемка
- История продаж и возвраты
- Дашборд и базовые отчеты

## 2. Что не входит в пилот

- Ресторанный режим
- Оффлайн-first продажи
- PWA как основной рабочий режим
- Сложные multi-cart сценарии кассира

## 3. Pre-Release Checklist

- [ ] Backend миграции применены через Alembic
- [ ] Создан администратор
- [ ] Проверены переменные окружения фронтенда
- [ ] Проверены переменные окружения backend
- [ ] Smoke test из `MVP_SMOKE_TEST_CHECKLIST_RU.md` пройден
- [ ] Подготовлен rollback-план

## 4. Настройки окружения

### Frontend

- `NEXT_PUBLIC_API_URL=/api`
- `NEXT_PUBLIC_API_PROXY_TARGET=http://127.0.0.1:8000`
- `NEXT_PUBLIC_ENABLE_RESTAURANT=false`
- `NEXT_PUBLIC_ENABLE_OFFLINE_MODE=false`

### Backend

- Настроить базу данных
- Применить миграции
- Убедиться, что приложение стартует без авто-создания схемы

## 5. Роль оператора

Оператор в пилоте должен уметь:

- войти в систему
- открыть кассу
- добавить товар в корзину
- завершить продажу
- найти продажу в истории
- оформить возврат
- принять поставку по закупке

## 6. Порядок запуска в точке

1. Поднять backend и проверить `/health`
2. Поднять frontend
3. Выполнить вход под тестовым пользователем
4. Проверить тестовую продажу
5. Проверить тестовую приемку закупки
6. Проверить историю продаж и отчеты

## 7. Поддержка во время пилота

- Если сервер недоступен, не продолжать продажи как будто они сохранены
- Если остатки расходятся, сверять последнюю приемку и последние продажи
- Если возврат не проходит, фиксировать sale id и текст ошибки

## 8. Rollback Notes

- Остановить frontend rollout
- Вернуть предыдущую стабильную сборку frontend
- Не откатывать базу без отдельного решения
- Сначала сохранить логи и идентификаторы проблемных продаж/закупок
