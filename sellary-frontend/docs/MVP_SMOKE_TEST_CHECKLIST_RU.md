# MVP Smoke Test Checklist

## 1. Login

- [ ] Открывается страница входа
- [ ] Пользователь `admin / admin123` успешно входит в систему
- [ ] После входа открывается экран кассы
- [ ] При невалидном пароле показывается ошибка

## 2. POS Checkout

- [ ] На кассе открывается список товаров через кнопку добавления
- [ ] Товар добавляется в корзину
- [ ] Количество увеличивается и уменьшается без ошибок
- [ ] Количество нельзя увеличить выше доступного остатка
- [ ] Итоговая сумма обновляется после изменения корзины
- [ ] Продажа наличными успешно завершается
- [ ] После успешной продажи корзина очищается

## 3. Stock Update

- [ ] После продажи остаток товара уменьшается
- [ ] При попытке продать больше, чем есть на складе, операция блокируется
- [ ] Ошибка по нехватке остатка показывается явно
- [ ] Товар с нулевым остатком нельзя добавить в продажу из каталога

## 4. Purchase Receive

- [ ] Открывается список закупок
- [ ] Можно создать новый черновик закупки
- [ ] Черновик можно отправить поставщику
- [ ] Для отправленной закупки открывается приемка
- [ ] После приемки остаток товара увеличивается

## 5. Sales History

- [ ] Новая продажа появляется в истории продаж
- [ ] Для продажи открывается детальная карточка
- [ ] Частичный возврат оформляется без ошибки
- [ ] После возврата статус и суммы в истории обновляются

## 6. Dashboard And Reports

- [ ] На дашборде отображаются продажи за день
- [ ] В блоке low stock видны товары с критичным остатком
- [ ] В отчетах отображается график продаж
- [ ] В отчетах отображается список топ-товаров

## 7. MVP Gate

- [ ] Раздел ресторана недоступен в MVP
- [ ] Оффлайн-синхронизация не предлагает рабочий сценарий продажи
- [ ] При выключенном сервере система не создает фиктивные продажи
