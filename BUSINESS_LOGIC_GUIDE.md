## 📌 Краткий обзор системы

**Sellary POS** — современная система для точек продаж, разработанная для **малого и среднего розничного бизнеса**. Система оптимизирует ежедневные операции магазина:

| Возможность | Ценность для бизнеса |
|-------------|---------------------|
| **Быстрое оформление продаж** | Сокращение времени обслуживания клиентов благодаря сканированию штрих-кодов и мульти-вкладкам |
| **Учёт товаров в реальном времени** | Всегда актуальные остатки; невозможно продать больше, чем есть |
| **Управление поставщиками** | Отслеживание закупок и автоматизация заказов |
| **Информативные отчёты** | Принимайте решения на основе данных о прибыли и продажах |
| **Разграничение доступа** | Безопасный доступ для администраторов, менеджеров и кассиров |

### 🎯 Целевые пользователи
- **Владельцы розничных магазинов** — модернизация бизнес-процессов
- **Менеджеры магазинов** — контроль склада и продаж
- **Кассиры** — оформление ежедневных транзакций

## 💰 Продажи и касса

### Как происходит продажа

Система POS обеспечивает **быстрое и надёжное оформление продаж**:

```
┌────────────────────────────────────────────────────────────────────┐
│                      ЖИЗНЕННЫЙ ЦИКЛ ПРОДАЖИ                        │
├────────────────────────────────────────────────────────────────────┤
│                                                                    │
│  [1] ДОБАВИТЬ     ──▶  [2] ПРОВЕРИТЬ   ──▶  [3] ОФОРМИТЬ ОПЛАТУ   │
│      В КОРЗИНУ             КОРЗИНУ                                 │
│        │                     │                      │              │
│        ▼                     ▼                      ▼              │
│   • Сканирование        • Изменить кол-во     • Выбрать способ    │
│     штрих-кода          • Удалить товар         оплаты            │
│   • Поиск по имени      • Просмотр итогов     • Завершить         │
│   • Проверка остатка                            продажу           │
│                                                                    │
└────────────────────────────────────────────────────────────────────┘
```

### Правила обработки продаж

| Правило | Описание | Влияние на бизнес |
|---------|----------|-------------------|
| **Проверка остатков** | Нельзя добавить в корзину больше, чем есть на складе | Защита от пересортицы |
| **Атомарная транзакция** | Все операции выполняются вместе или не выполняются совсем | Целостность данных |
| **Автоматическое списание** | Остаток товара уменьшается сразу после продажи | Точность учёта в реальном времени |
| **Аудит** | Каждая продажа создаёт записи в журнале склада | Полная прослеживаемость |

### Поддерживаемые способы оплаты

| Способ | Варианты | Дополнительно |
|--------|----------|---------------|
| 💵 **Наличные** | Стандартная оплата наличными | Без дополнительных полей |
| 💳 **Карта** | Алиф Банк, Эсхата Банк, DC (Душанбе Сити) | Тип карты сохраняется для отчётов |
| 📱 **Мобильный платёж** | Мобильные платёжные системы | Готовность к интеграции |

### Статусы продажи

```
                    ┌────────────────┐
                    │   ЗАВЕРШЕНА    │ ◀── Обычная продажа
                    └───────┬────────┘
                            │
           ┌────────────────┼────────────────┐
           │                │                │
           ▼                ▼                ▼
    ┌──────────────┐ ┌────────────────┐ ┌──────────────┐
    │   ОТМЕНЕНА   │ │   ЧАСТИЧНЫЙ    │ │   ВОЗВРАТ    │
    │   (Полная)   │ │    ВОЗВРАТ     │ │   (Полный)   │
    └──────────────┘ └───────┬────────┘ └──────────────┘
                             │
                             ▼
                     ┌──────────────┐
                     │   ВОЗВРАТ    │
                     └──────────────┘
```

| Статус | Значение | Влияние на склад | Можно отменить? |
|--------|----------|------------------|-----------------|
| **Завершена** | Обычная продажа | Товар списан | Да (отмена или возврат) |
| **Отменена** | Продажа аннулирована | Товар возвращён на склад | Нет (конечный статус) |
| **Частичный возврат** | Некоторые товары возвращены | Частичный возврат на склад | Можно завершить возврат |
| **Возврат** | Все товары возвращены | Полный возврат на склад | Нет (конечный статус) |

### Мульти-вкладки (Параллельные сессии)

**Бизнес-потребность:** Кассиру часто нужно приостановить обслуживание одного клиента для обслуживания другого (клиент ищет кошелёк, проверка цены и т.д.)

**Как это работает:**
- Кассир может открыть **несколько сессий корзины** (вкладок)
- Переключение между вкладками без потери товаров
- Каждая вкладка сохраняет своё состояние корзины
- Продажи оформляются независимо по каждой вкладке

### Горячие клавиши

| Клавиша | Действие |
|---------|----------|
| `F2` | Открыть поиск товаров |
| `Enter` | Завершить продажу (когда открыто окно оплаты) |

## 📦 Управление складом

### Правила движения товаров

**Приход (Увеличение остатка):**
| Источник | Триггер | Описание |
|----------|---------|----------|
| Приёмка заказа поставщику | Отметить товары как полученные | Товар прибыл от поставщика |
| Отмена продажи | Отменить завершённую продажу | Возврат товара от аннулированной продажи |
| Возврат от покупателя | Оформить возврат | Клиент вернул товар(ы) |
| Ручная корректировка | Действие администратора | Пересчёт, найденный товар |

**Расход (Уменьшение остатка):**
| Источник | Триггер | Описание |
|----------|---------|----------|
| Завершение продажи | Оформление чека | Товары проданы клиенту |
| Ручная корректировка | Действие администратора | Бой, недостача, образцы |

### Оповещения о низком остатке

**Как это работает:**
1. Для каждого товара задаётся **Минимальный остаток** (настраивается)
2. Система постоянно сравнивает текущий остаток с минимальным
3. Товары с остатком ниже минимума отображаются в **предупреждениях на главной панели**
4. Отображается в виджете **«Товары с низким остатком»**

**Пример:**
```
Товар: "Премиум Кофе 500г"
Текущий остаток: 3 шт.
Минимальный остаток: 10 шт.
→ ⚠️ Предупреждение: "Премиум Кофе 500г требует пополнения!"
```

### Журнал склада (Аудит)

Каждое изменение остатка фиксируется навсегда:

| Поле | Назначение |
|------|------------|
| Товар | Какой товар был затронут |
| Изменение количества | +5, -2 и т.д. |
| Предыдущий остаток | Остаток до изменения |
| Новый остаток | Остаток после изменения |
| Причина | "Продажа #1234", "Корректировка", "Заказ поставщику #567" |
| Пользователь | Кто внёс изменение |
| Дата и время | Когда это произошло |

**Ценность для бизнеса:** Полная прозрачность и ответственность при расхождениях в инвентаризации.

### Приёмка товаров (Поддержка частичной поставки)

**Бизнес-сценарий:** Поставщик привёз 50 единиц, а вы заказывали 100.

**Как это работает:**
1. Откройте «Принять товары» для заказа
2. Введите фактически полученное количество по каждой позиции
3. Система обновит:
   - Остаток товара (увеличится на принятое количество)
   - Счётчик «Получено» для позиции заказа
   - Статус заказа (ЧАСТИЧНО ПОЛУЧЕН если не всё, ПОЛУЧЕН если всё)
4. Создаются записи в журнале склада со ссылкой на заказ

**Пример:**
```
Заказ №1001 — Позиции:
├── Товар А: Заказано 100 → Получено 50 → Статус: Частично
├── Товар Б: Заказано 20  → Получено 20 → Статус: Полностью
└── Товар В: Заказано 30  → Получено 0  → Статус: Ожидается

Общий статус заказа: ЧАСТИЧНО ПОЛУЧЕН
```

## 🧮 Финансовые расчёты

### Расчёт цен и налогов

Система использует точные десятичные вычисления для всех финансовых операций.

**Для каждой позиции продажи:**
```
Подытог          = Количество × Цена за единицу
Сумма налога     = Подытог × (Процент налога ÷ 100)
Итого по позиции = Подытог + Сумма налога - Скидка
```

**Для всей продажи (чека):**
```
Подытог продажи = Сумма подытогов всех позиций
Налог продажи   = Сумма налогов всех позиций
Итого продажи   = Подытог продажи + Налог продажи - Скидка на продажу
```

### Расчёт прибыли

**Для каждого товара:**
```
Прибыль    = (Цена продажи - Себестоимость) × Проданное количество
Маржа (%)  = ((Цена продажи - Себестоимость) ÷ Себестоимость) × 100
```

**Пример:**
```
Товар: "Чехол для ноутбука"
├── Себестоимость:      150.00 сомони
├── Цена продажи:       250.00 сомони
├── Продано:            10 шт.
└── Расчёты:
    ├── Прибыль с единицы: 250.00 - 150.00 = 100.00 сомони
    ├── Общая прибыль:     100.00 × 10 = 1000.00 сомони
    └── Маржа:             (100.00 ÷ 150.00) × 100 = 66.67%
```

### Оценка стоимости склада

Общая стоимость склада рассчитывается как:
```
Стоимость склада = Σ (Остаток товара × Себестоимость товара)
```

Это представляет **себестоимость** вашего текущего склада.

## 📊 Отчёты и аналитика

### Виджеты главной панели

Панель управления показывает ключевые бизнес-показатели:

| Виджет | Отображаемые данные | Обновление |
|--------|---------------------|------------|
| **Продажи сегодня** | Общая выручка за текущий день | В реальном времени |
| **Прибыль сегодня** | Выручка минус себестоимость за сегодня | В реальном времени |
| **Количество чеков** | Число продаж сегодня | В реальном времени |
| **Низкий остаток** | Товары ниже минимального остатка | В реальном времени |
| **Топ продаж** | Топ-5 по количеству за сегодня | В реальном времени |
| **Последние продажи** | 10 последних транзакций | В реальном времени |

### Доступные отчёты

#### 📈 Отчёт по продажам
- **Назначение:** Отслеживание трендов выручки
- **Данные:** Дневные продажи за выбранный период
- **Визуализация:** Линейный график дневной динамики
- **Периоды:** 7 дней, 30 дней, 90 дней, 1 год

#### 💵 Отчёт по прибыли
- **Назначение:** Понимание рентабельности
- **Показатели:**
  - Общая выручка (сумма продаж)
  - Общая себестоимость (стоимость товаров)
  - Валовая прибыль (выручка - себестоимость)
  - Процент маржи
- **Визуализация:** Столбчатая диаграмма сравнения выручка vs себестоимость vs прибыль

#### 🏆 Топ товаров
- **Назначение:** Определение бестселлеров
- **Данные:** Товары, отсортированные по количеству продаж
- **Включает:** Название товара, продано единиц, выручка, прибыль
- **Визуализация:** Горизонтальная столбчатая диаграмма

## 🔄 Схемы бизнес-процессов

### Полный процесс продажи

```
┌─────────────────────────────────────────────────────────────────────┐
```

## ❓ Часто задаваемые вопросы

### Продажи и касса

**В: Может ли кассир продать больше товаров, чем есть на складе?**
> Нет. Система проверяет остатки при оформлении. Если товара недостаточно, продажа будет заблокирована.

**В: Что произойдёт, если я отменю продажу?**
> Продажа будет помечена как «Отменена», и весь товар автоматически вернётся на склад. Это действие нельзя отменить.

**В: Можем ли мы сделать раздельную оплату (часть наличными, часть картой)?**
> В данный момент нет. Каждая продажа требует один способ оплаты. Это возможное улучшение в будущем.

**В: Как обработать возврат от покупателя?**
> Используйте функцию «Отменить продажу» или «Возврат». Товар автоматически вернётся на склад, статус продажи обновится.

---

### Склад

**В: Что делать, если фактический остаток не совпадает с системой?**
> Используйте функцию «Ручная корректировка остатка» (только для Админа/Менеджера). Введите правильное количество с причиной (например, «Пересчёт — Апрель 2026»).

**В: Можно ли увидеть, кто изменил остаток?**
> Да. Журнал склада показывает каждое изменение, включая пользователя, дату/время и причину.

**В: Как узнать, когда нужно заказать товар?**
> Установите «Минимальный остаток» для каждого товара. Когда остаток упадёт ниже этого уровня, товар появится в виджете «Низкий остаток» на главной панели.

---

### Поставщики и закупки

**В: Можно ли принять частичную поставку от поставщика?**
> Да. Функция «Принять товары» поддерживает частичную приёмку. Статус заказа покажет «Частично получен» до полного получения всех товаров.

**В: Что если поставщик привёз больше, чем мы заказали?**
> Система позволяет принять только до заказанного количества по каждой позиции. Лишний товар нужно отследить отдельно или обновить заказ.

**В: Можно ли удалить заказ поставщику?**
> Только если он в статусе «Черновик». После отправки его можно только отменить (не удалить) — для целей аудита.

---

### Отчёты и аналитика

**В: Как рассчитывается прибыль?**
> Прибыль = Выручка (сумма продаж) - Себестоимость (закупочная цена × количество). Система автоматически рассчитывает это из себестоимости товаров.

**В: Можно ли экспортировать отчёты в Excel или PDF?**
> В текущей версии нет. Это запланировано на будущие релизы.

**В: Как часто обновляются данные на главной панели?**
> В реальном времени. Каждая продажа, отмена или изменение склада мгновенно обновляет показатели.

## 📝 Приложение: Глоссарий

| Термин | Определение |
|--------|-------------|
| **POS** | Point of Sale (Точка продажи) — интерфейс кассы |
| **SKU** | Stock Keeping Unit — уникальный идентификатор товара |
| **Штрих-код** | Сканируемый код товара (EAN, UPC и др.) |
| **Себестоимость** | Цена закупки товара у поставщика |
| **Цена продажи** | Цена, которую платит покупатель |
| **Маржа** | Разница между ценой продажи и себестоимостью (прибыль) |
| **Заказ поставщику** | Заявка на закупку товаров |
| **Себестоимость проданных товаров** | Общая стоимость закупки проданных товаров |
| **Низкий остаток** | Количество товара ниже настроенного минимума |
| **Журнал склада** | Полная история всех движений товара |
| **Чек** | Документ, подтверждающий продажу |
| **Инвентаризация** | Физический пересчёт товаров на складе |
